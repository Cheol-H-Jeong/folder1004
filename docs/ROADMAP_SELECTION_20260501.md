# Folder1004 발전 아이디어 20개와 이번 구현 선정

## 후보 20개
1. 폴더 프로필 자동 감지: Downloads/사진/수업/업무/연구/코드 폴더 성격을 로컬 휴리스틱으로 식별.
2. 폴더 건강 점수: 루트에 방치된 파일, 임시/설치 파일, 중복 의심, 오래된 파일 비율로 정리 필요도를 수치화.
3. 프로필 기반 분류 스타일 추천을 공용 모듈화: UI와 파이프라인이 같은 추천 로직 사용.
4. Dry-Run 리포트에 “왜 이렇게 분류했는지” 더 풍부한 설명 추가.
5. 정리 전 위험 파일 감지: 실행 파일/숨김 파일/대용량 파일 별도 경고.
6. 폴더별 프리셋 히스토리: 같은 폴더를 다시 열면 지난 스타일 복원.
7. 리포트 요약 카드 개선: 비용/속도/건강 점수/프로필 한눈에 보기.
8. Mock 분류 품질 향상: 파일명 키워드 + 날짜 + 폴더 프로필 반영.
9. 결과 폴더명 미리보기: 이동 전 생성될 폴더명만 보여주기.
10. 되돌리기 매니페스트: 실제 이동 후 원래 위치 복원용 manifest 생성.
11. 검색 결과 필터: 확장자/기간/카테고리 필터.
12. 로그 뷰 필터: 오류/경고/LLM/파일 이동별 필터.
13. 폴더별 제외 패턴 UI: 특정 확장자/이름 제외를 시작 화면에서 편집.
14. 대용량 폴더 샘플링 미리 분석: 전체 파싱 전 빠른 프로필/비용 추정.
15. 이미지/영상 EXIF 날짜 우선 분류.
16. ZIP 내부 대표 파일명 기반 분류 강화.
17. Windows/macOS 권한 문제 사전 진단.
18. LLM 응답 JSON repair 상세 리포트.
19. 정리 후 빈 폴더 정리 후보 리포트.
20. 시나리오 테스트용 샘플 폴더 생성 CLI.

## 이번에 현실적으로 선정한 기능
선정 기준: credentials/payment/external production 불필요, 로컬에서 결정 가능, 기존 아키텍처에 작은 변경으로 통합 가능, 테스트 가능.

- 1. 폴더 프로필 자동 감지
- 2. 폴더 건강 점수
- 3. 프로필 기반 추천 로직 공용 모듈화의 기반
- 7. 리포트 요약 카드 개선 중 Markdown 리포트 반영
- 14. 빠른 프로필/정리 필요도 추정의 파이프라인 기반

이번 범위 밖: 실제 복원 기능(파괴적 이동과 맞물려 별도 안전 설계 필요), 외부/OS 백그라운드 tray, credentials 필요한 LLM 신규 기능.
